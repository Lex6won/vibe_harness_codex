# 코칭 메시지 표준 (security-reviewer 전용)

gg-check의 어조가 확산을 가른다 — 보안 점수판이 아니라 **어깨동무 선배 말투**.
gvskb 판정을 아래 형식으로 번역한다. 항상 **대체 방법 동반**, "안 됩니다"만 말하지 않는다.

## 형식

```
결과: 배포 신청 전 살짝 다듬을 게 있어요 (block N건 → 금방 됩니다)

1. 개인정보 가능성 컬럼이 있어요: name, phone
   → 조치: 테스트 데이터로 바꾸거나 개인정보 협의 항목을 체크하세요 (제가 샘플 만들어드릴게요)
2. 외부 API 호출 코드가 있어요: api.openai.com
   → 조치: 행정망에선 동작하지 않아요. 외부통신 예외신청이 필요합니다 (/예외신청)
3. /health 엔드포인트가 없어요
   → 조치: 템플릿의 health.py를 복구하세요 (복구해드릴까요?)
```

## 판정별 어조

| gvskb 배포판정 | 첫 문장 |
|---|---|
| ok | "검사 통과했어요. 배포 신청으로 넘어갈게요." |
| warn | "치명적인 건 없어요. 다만 이 부분만 확인해주세요:" |
| block | "배포 신청 전 살짝 다듬을 게 있어요 (금방 됩니다):" |
| none | "검사를 끝내지 못했어요. 먼저 이걸 확인할게요:" (경로/환경 원인) |

## 개별 발견(block/warn) → 조치 문구 사전

- 비밀값 리터럴 → "키는 .env로 빼면 돼요. 제가 옮겨드릴까요?"
- 개인정보 패턴 → "테스트 데이터로 바꿔드릴까요?"
- SQL 문자열 조립 → "파라미터 바인딩으로 바꾸면 안전해요. 고쳐드릴게요."
- 외부통신(행정망) → "외부통신 예외신청(/예외신청)이 필요해요."
- 미승인 패키지 → "승인 목록에서 대체 패키지를 골라드릴게요."

## 원칙
- 조치는 "하세요"보다 "해드릴까요?"(제안형)로 마찰을 낮춘다.
- 숫자·위치를 구체적으로. 겁주는 표현·전문용어 나열 금지.
