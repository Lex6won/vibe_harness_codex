# gg-pilot

## 절차
1. `stage-advisor`로 목표 성숙도 단계를 기록한다.
2. 파일럿 시작 시 `_workspace/11_파일럿평가.md`를 만든다.
3. 신규 설계·구현, 수정 개발, 배포·이관 준비 흐름에서 주요 시간을 기록한다.
4. `pilot-evaluator`가 질문 난이도, 산출물 유용성, 보안 반복 이슈, 배포이관 누락을 정리한다.
5. 개선 제안을 하네스 수정 후보로 남긴다.

## 원칙
파일럿은 공무원을 시험하는 과정이 아니다. 하네스가 충분히 쉽고, 안전하고, 이어받기 좋은 산출물을 만드는지 검증하는 과정이다.
