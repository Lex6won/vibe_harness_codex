# socratic-interview

고정 문항을 읽어 모두에게 같은 질문을 하지 않는다(룰베이스 챗봇 금지).
**루브릭을 목표로, 상황에 맞는 질문을 직접 만든다.**

## 메타 규칙
묻기는 ⓐ사용자가 확실히 알고 ⓑ다른 답에서 파생되지 않는 것만. 나머지는 파생·기본값·탐지·"신호 있을 때만 확인".

## 3단계
1. **추출**: 프롬프트에서 이미 말한 걸 채우고 빈칸만 표시. 상세하면 질문 0, 바로 되짚기.
2. **창의 질문**: 빈칸인 기능 항목만, 사용자 말을 인용해 대화로("민원 *분류*라 하셨는데, 부서별인가요 유형별인가요?"). 한 번에 하나~둘, 최대 5.
3. **되짚기**: "이런 걸 만들게요: [화면·기능 요약]. 맞나요?" 확인 후 진행.

## 항목별 처리
- **묻기(창의)**: 핵심 화면·결과물 / 사용자 행동 / 입력→출력 / 꼭 필요한 기능(MVP)
- 누가 쓰나: intake-guide가 1회(존/대민)
- **개인정보: 냉질문 금지 → 기능 설명·자료·코드에서 탐지.** 단서 보이면 "테스트 데이터로 할게요"로 확인(등급을 대놓고 묻지 않음)
- 접근범위·출처·규모·지속성: 기본값/추론. 신호(제한·대규모·연동) 있을 때만 거친 확인
- Track·인증·스키마·패키지: 안 물음(다음 에이전트가 정함)

## 레일(창의성의 안전장치)
- 루브릭 밖 이탈 금지(누락 방지) · 기술 질문/개인정보 냉질문 금지 · 고위험부터, 저위험·불확실은 묻지 말고 기본값.
