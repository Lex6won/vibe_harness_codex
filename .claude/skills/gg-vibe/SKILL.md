# gg-vibe

이 스킬은 Codex 호환 wrapper다. 실제 canonical workflow는 `../gg-vibecode/skill.md`를 따른다.

## 라우팅
- 새로 만들기: `gg-vibecode` 신규 설계·구현 모드.
- 기존 코드 수정: `gg-change` 또는 `gg-vibecode` 기존 코드 수정 모드.
- 배포/이관/보안성검토: `gg-release` 또는 `gg-vibecode` 배포·이관 준비 모드.
- 보안검사만: `gg-check`.
- 파일럿/실증 평가: `gg-pilot`.

## 성숙도 기준
- L0: 아이디어 구체화.
- L1: 내부 시제품. 사용자가 모르면 기본값.
- L2: 내부도구. 실제 업무 사용 후보.
- L3: 정식 서비스 후보. 배포·공식 개발환경 이관 준비.
- L4: 정식 운영. 하네스 단독 승인 판정 금지.

## 공통 원칙
- 공무원에게 기술결정을 묻지 않는다.
- 화면, 기능, 입력, 처리, 출력, 저장 여부는 반드시 확보한다.
- 운영환경, 패키지 정책, 행정망/외부망 제약을 지킨다.
- 개발 완료, 제출 준비, 승인 완료를 구분한다.
- 모든 변경은 `_workspace/00_작업현황.md`와 manifest에 남긴다.
