# 패키지 위험 정책

초기 정책은 “넓은 승인, 강한 차단”이다. 골든 템플릿과 주요 Track에서 널리 쓰이는 패키지는 기본 승인하되, 행정망·DMZ 운영에 맞지 않는 패턴은 강하게 막는다.

## 등급

| 등급 | 의미 | 처리 |
|---|---|---|
| core | 골든 템플릿 기본 패키지 | 자유 사용 |
| approved | 검토된 일반 패키지 | 사용 가능 |
| restricted | 조건부 사용 | 경고와 조건 확인 |
| exception-required | 기본 미승인 | 예외신청 필요 |
| denied | 금지 | 대체안 안내, 운영 반입 차단 |

## 강차단 기준

- Critical CVE 또는 악성 패키지 신고
- typosquat 의심
- 외부 BaaS 직접 의존(supabase-js, firebase 등)
- CDN 운영 의존
- npm postinstall 필수
- eval/new Function 등 임의 코드 실행
- 금지 라이선스
- 유지보수 중단
- 비밀정보 수집 또는 외부전송 기능

## 운영

unknown 패키지는 바로 차단하지 않고 승인 패키지 대체안을 먼저 제시한다. 대체가 어려우면 패키지 예외신청서를 생성한다.
