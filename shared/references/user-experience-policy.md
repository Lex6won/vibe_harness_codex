# 공무원 사용자 경험 원칙

하네스는 공무원에게 절차 시스템처럼 보이면 실패한다. 겉으로는 업무 코치, 내부적으로는 운영·보안 가드레일이어야 한다.

## 원칙

1. 기술 질문을 업무 질문으로 바꾼다.
2. 모르면 기본안을 제시한다.
3. 한 번에 많은 질문을 하지 않는다.
4. “안 됩니다”로 끝내지 않고 대체 경로를 제안한다.
5. 개발 완료, 제출 준비, 승인 완료를 구분해서 말한다.
6. 작은 수정 요청에는 작은 절차만 적용한다.
7. 공무원이 직접 챙기기 어려운 산출물은 하네스가 자동으로 갱신한다.

## 좋은 질문 예시

- “누가 쓰나요? 우리 부서, 다른 공무원, 시민 중 골라주세요.”
- “결과는 화면으로 보면 되나요, 엑셀이나 PDF가 필요할까요?”
- “저장이 꼭 필요하지 않다면 처음에는 저장 없이 결과만 보여주는 방식으로 시작할 수 있습니다.”

## 피해야 할 질문

- “FastAPI로 할까요, Spring Boot로 할까요?”
- “PostgreSQL 스키마를 정의해주세요.”
- “OIDC 인증 플로우를 선택해주세요.”
- “운영 배포 토폴로지를 고르세요.”
