# 공공 바이브코딩 파일럿 평가 기준

하네스 효과는 코드 품질만으로 평가하지 않는다. 공무원이 쉽게 시작했고, 운영·보안팀이 이어받을 수 있는 증거가 남았는지를 본다.

## 권장 파일럿 구성

- 내부 업무 보조 도구 3개
- 엑셀 업로드·집계형 도구 2개
- 간단한 대시보드 2개
- 파일업로드가 있는 위험 사례 1개
- 대민 서비스 후보 1개는 정식 배포가 아니라 전환 실험으로 수행

## 핵심 지표

| 지표 | 의미 | 수집 방법 |
|---|---|---|
| 첫 결과물 도달 시간 | 활성화 가능성 | 시작부터 L1 결과물까지 시간 |
| 질문 난이도 | 공무원 친화성 | 사용자가 어려웠던 질문 기록 |
| 산출물 유용성 | 운영팀/보안팀 연계성 | PRD, DB, 개발스택, 검증보고서 검토 의견 |
| 반복 보안 이슈 | 하네스 예방력 | checker block/warn 유형 |
| 수정 개발 소요 | 실사용 지속성 | 작은 수정 요청 처리 시간 |
| 배포·이관 누락률 | 정식화 가능성 | L3 패키지 누락 항목 수 |

## 성공 기준 예시

- 비개발 공무원이 1시간 안에 L0/L1 산출물을 얻는다.
- 내부도구 후보의 70% 이상이 승인 Track과 패키지 정책을 따른다.
- 배포·이관 준비 시 필수 산출물 누락이 기존 대비 줄어든다.
- 수정 개발 요청에서 전체 재설계로 돌아가는 비율이 낮다.
